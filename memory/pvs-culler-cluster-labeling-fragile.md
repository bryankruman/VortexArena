The §12.8 `WorldPvsCuller` hides whole world cells (coarse 1024-unit Godot grid in `MapLoader.RegroupIntoCells`; finalrage is only ~12 cells total — but culls 1–10 of them per viewpoint) by recording, per cell, the set of PVS clusters its geometry occupies, then showing a cell iff ANY is visible from the camera's cluster.

**History — the cluster labels used to be point-sampled (`BspPvs.FindLeaf(centroid)`), which caused TWO false-negative bugs (geometry winks out to skybox):**
1. **Axis swap** (fixed 08459a5): fed Godot-space centroid to the Quake-space `FindLeaf`. Must `Coords.ToQuake` first.
2. **Centroid-on-boundary** (fixed 2026-06-14): a triangle centroid lies exactly ON its surface = a leaf boundary; `FindLeaf` (`d>=0`→front) resolved ~10% into the solid leaf (−1) or the wrong side, so a cell missed the clusters its faces are visible FROM. Repro: finalrage +X corridor past quake X=1024, viewed from the −161,216 spawn.

**Current design (2026-06-14, replaces the sampling entirely):** labels are now **DP's exact per-surface vis** — `BspPvs.BuildFaceClusterSets()` unions the clusters of every leaf that references a face via **lump-5 leaffaces** (`BspData.LeafFaces`, now parsed by `BspReader.ReadLeafFaces`). `SurfaceBuilder.TriangleFaces` tags each triangle with its source face (appended in `AppendPolygonFace`/`AppendTessellation`); `RegroupIntoCells` unions `faceClusters[face]` per triangle. Validated on finalrage: **0/2912** world faces uncovered (vs 10.5% of centroids landing in solid). No sampling → the whole bug class is gone. The nudge heuristic + `PvsClusterNudge` const were removed. Tests: `BuildFaceClusterSets_Unions_Every_Referencing_Leafs_Cluster` + real-data `Finalrage_Corridor_Cell_Is_Visible_From_The_Reported_Spawn` in BspPvsTests.cs.

**Still fundamentally COARSE:** per-cell union at 1024-unit granularity. If a future map needs tighter occlusion, refine cell size or go per-leaf/per-surface.

**DP-faithful entity render culling (2026-06-14, `r_pvs_cull_entities`, default 0):** DP culls entities (networking + render) by PVS, NOT a geometric occluder — and we only used `BspPvs` for bot LOS, not entity render. `ClientWorld.ApplyEntityPvsCull` now hides remote `EntityNode`s whose BOUNDS (origin ± `r_pvs_cull_entities_margin`, default 64) are outside the camera's PVS, via `BspPvs.BoxAnyClusterVisibleFrom` (DP `Mod_Q3BSP_BoxTouchingPVS` — box test, not point, so a model peeking past a boundary can't wink out). Local player never culled; unvised map / camera-in-solid → show all. Visibility is a dual flag on EntityNode (`SetGameplayVisible`/`SetPvsVisible` AND together) so it doesn't fight the item ghost/fade writers. `NetGame` wires `_render.Pvs`. **Default ON (1) since 2026-06-14** (verified: items in view render identical with cull on vs off; set 0 to disable for A/B).

**DP-faithful NETWORKING entity cull (2026-06-14, `sv_cullentities_pvs`, default 1 = DP default):** the server now stops SENDING a client entities outside its PVS (DP `SV_MarkWriteEntityStateToClient`) — the bandwidth + draw-call win. `ServerNet.RelevantEntitiesFor` filters `_entityScratch` per recipient (box-test each entity's bounds via `BoxAnyClusterVisibleFrom`; bounds precomputed in `BuildEntitySet` → `_entityBounds`, `RelevanceBounds` = hull ∪ ±48 floor). KEY: the snapshot delta encoder ALREADY handles appear/disappear — a filtered-out entity becomes a "removed" vs that client's baseline, a re-entering one a "spawn" with current state (NOT the delicate-baseline trap I feared; locked by `Snapshot_Culled_Entity_Removes_Then_Respawns_With_Current_State`). Conservative fallbacks: unvised map / dead-or-spectating recipient / eye-in-solid → send all. NOT done (DP has, off-by-default there too): `sv_cullentities_trace` (LOS traceline cull + anti-flicker delay + prediction). Note: inline `*N` bmodels aren't networked here (map geometry), so DP's nevercullbmodels is moot.

**Adaptive WorldCellSize (2026-06-14, `r_world_cell_*`, default off):** `MapLoader.ResolveCellSize` — `r_world_cell_adaptive 1` scales the §12.5 cell size to map extent (`maxAxis/r_world_cell_div`, clamped `_min`/`_max`) so cell COUNT (≈draw calls) stays bounded; a fixed small size would regress large/open maps. adaptive 0 → fixed `r_world_cell_size` (1024 = today). Measured: finer chunks raise tri-cull (stormkeep 18%→33% at 512) but draw calls explode and saturate ~256-512; large OPEN maps barely benefit (implosion 12%→16%). Read at map LOAD (needs reload). finalrage adaptive → cellSize 304 / 166 cells (was 12).

**Godot-native occlusion culling A/B scaffold (2026-06-14, `game/WorldOcclusion.cs`):** an orthogonal alternative behind cvar **`r_occlusion_cull`** (default 0; independent of `r_pvs_cull` — the 4 A/B combos are the 2×2). `MapLoader.BuildWorldOccluder` merges the OPAQUE world surfaces into one `ArrayOccluder3D` (excludes translucent/liquid/nonsolid/sky via `IsOpaqueOccluder`); `WorldOcclusion` toggles it + `Viewport.UseOcclusionCulling` at runtime. Project setting `rendering/occlusion_culling/use_occlusion_culling=true` added (allocates the buffer; per-viewport flag still gates it). **Occluder is the render mesh, NOT brushes** — finalrage has 17.6k bevel-padded solid brushes (hundreds of k tris) vs the render mesh's ~102k tris (incl. tessellated patches). A/B it: `--cvar r_occlusion_cull 1 --cvar r_pvs_cull 0` (verified: loads + renders finalrage with no over-cull). Caveat for tighter culling: the ~12 cells are 1024³ — their AABBs are too big for Godot's AABB-occlusion test to bite well; needs smaller render chunks to really benefit (the §12.5 split).

**How to diagnose against a real BSP:** parse finalrage.bsp straight from `assets/data/xonotic-20230620-maps.pk3` in Python (header→17 lumps; planes=16B, nodes=36B[plane,c0,c1], leafs=48B[cluster@0, firstLeafFace@32, numLeafFaces@36], leaffaces=lump5 int32[], verts=44B[pos@0], meshverts=lump11 int32[], faces=104B[type@8, vtx/nvtx/mv/nmv@12, normal@84], vis@16). Build face→clusters from leaffaces, replicate cell-binning (`godot=(qx,qz,-qy)`, cell=floor(godot/1024)), check `ClustersVisible(camCluster, c)`. **HUD position readout is QUAKE coords** — confirm via `info_player_deathmatch` origins in the entity lump. See [[camera-drift-render-smoothing]] for the other render-layer caveats.
