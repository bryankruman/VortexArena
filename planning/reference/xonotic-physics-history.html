Complete physicsX.cfg values at xonotic-v0.8.0

Every cvar explicitly set in physicsX.cfg at the xonotic-v0.8.0 tag, confirmed with git show xonotic-v0.8.0:physicsX.cfg. The "Engine" column is the DarkPlaces C-code cvar_t default that Xonotic overrides at startup.

Cvar Engine default Xonotic 0.8.0 Notes
Ground movement
sv_gravity 800 800 Identical to engine default. Visited 700, then 720 (2013–14) before being restored — see timeline.
sv_maxspeed 320 360+12.5% Ground run speed. Nexuiz default was ~400 u/s.
sv_maxairspeed 30 360×12 Engine default of 30 effectively disables air techniques; Xonotic matches ground speed.
sv_stopspeed 100 100 Speed threshold below which friction halts movement.
sv_friction 4 6+50% Ground deceleration. Was 8 pre-2013; Samual reduced to 6 in Aug 2013.
edgefriction 2 1 Friction multiplier near ledge edges.
sv_accelerate 10 15+50% Ground acceleration rate toward sv_maxspeed.
sv_stepheight 18 31 Max step-up height in Quake Units. Was 26 pre-2013; Samual raised it Aug 2013. File comment: "just below 32, keeping things smooth without allowing 32qu steps."
Jump
sv_jumpvelocity 270 260 Jump impulse (u/s). File comment: "actually 266.6666 for 180bpm, but 260 is nicer to mappers." At 800g → 42.25 QU apex height; just clears a 112 QU corridor. Had been 245→248→260 between 2013–2014.
sv_doublejump 0 0 Q2-style double jump; explicitly disabled.
sv_jumpspeedcap_min — "" Jump speed floor (empty = inactive). Changed to "nan" in Nov 2025 for float networking.
sv_jumpspeedcap_max — "" Jump speed ceiling (empty = inactive). Ditto Nov 2025.
sv_jumpspeedcap_max_disable_on_ramps 0 1 Disables the cap on ramps to preserve ramp-jump behaviour.
sv_friction_on_land — 0 Extra friction burst on landing; disabled.
Water & slick surfaces
sv_wateraccelerate 10 −1 −1 inherits sv_accelerate.
sv_waterfriction 4 −1 −1 inherits sv_friction.
Air acceleration
sv_airaccelerate −1 2 Engine −1 inherits sv_accelerate; Xonotic sets a small explicit value to constrain air control.
sv_airaccel_sideways_friction 0 0 Friction when strafing sideways in air; no penalty.
sv_airaccel_qw 1 −0.8 Not 1 (not unchanged from engine). Negative value inverts QW acceleration character. Engine's 1 = full QW bunny accel. Xonotic's −0.8 suppresses it significantly.
sv_airaccel_qw_stretchfactor 0 2 Stretches the effective QW accel envelope.
sv_airstopaccelerate 0 3 Not 0 (not disabled). Separate deceleration rate when reversing direction in air.
sv_airstrafeaccelerate 0 18 Not 0 (not disabled). Was 24 pre-2013; Samual reduced it Aug 2013.
sv_maxairstrafespeed 0 100 Speed cap when strafing in air.
sv_airstrafeaccel_qw 0 −0.95 QW ratio for strafe acceleration; negative inverts direction.
Air control (CPMA-style)
sv_aircontrol 0 100 CPMA-style mid-air steering. Was 125 pre-2013; Samual reduced to 100 Aug 2013.
sv_aircontrol_penalty 0 0 Speed penalty for using aircontrol. Was 180 pre-2013; Samual removed it Aug 2013.
sv_aircontrol_power 2 2 Exponent for the aircontrol steering curve.
sv_airspeedlimit_nonqw — 900 Soft speed cap for non-QW air movement. Was 800 pre-2013; Samual raised to 900 Aug 2013.
Warsow-style bunny hop (disabled at default)
sv_warsowbunny_turnaccel 0 0 Warsow turn accel; 0 disables the entire subsystem.
sv_warsowbunny_accel 0 0.1593 Warsow accel multiplier — inert while turnaccel=0.
sv_warsowbunny_topspeed 0 925 Warsow max bunny speed — inert while turnaccel=0.
sv_warsowbunny_backtosideratio 0 0.8 Warsow back-to-side ratio — inert while turnaccel=0.
Teleporter
g_teleport_maxspeed — 0 Max speed preserved through a teleporter (0 = unlimited).
Server-side: xonotic-server.cfg (not physicsX.cfg)
sv_maxvelocity 2000 10⁹ Effectively uncaps total velocity. Remains in xonotic-server.cfg.
sv_gameplayfix_q2airaccelerate 0 1 Enables Q2-style air acceleration formula. Remains in xonotic-server.cfg.
Corrections from prior analysis Three values in earlier summaries were wrong because they were derived without git access: sv_airaccel_qw was listed as 1 (unchanged from engine) — it is −0.8; sv_airstopaccelerate as 0 (disabled) — it is 3; sv_airstrafeaccelerate as 0 (disabled) — it is 18. Additionally, ~14 cvars that physicsX.cfg explicitly sets were omitted entirely — all are present above.

2013-08-15 5a30545
Samual's major overhaul — eight cvars changed in one commit

Samual Lenks switches physicsX.cfg to the values from his personal preset (physicsSamual.cfg). Commit message: "Switch my physics to be default for testing purposes." This is the largest single set of changes in the file's history. Lower gravity paired with a proportionally lower jump velocity preserves approximately the same jump height; the friction reduction makes movement feel less sticky.

2022 → xonotic-v0.8.5
Player physics migrated from DarkPlaces C code to QuakeC

Release notes: "Player physics is now implemented in QuakeC gamecode instead of using the engine physics." physicsX.cfg cvar names and values are unchanged — the code that reads them moved from C to QC. QC uses PHYS_* macro wrappers (PHYS_AIRACCELERATE, etc.) over the same cvars. One physicsX.cfg addition in this release era:

Remaining open questions

? QC vs. engine behavioural parity for complex cvars. The 0.8.5 migration preserved cvar names and values but moved the interpreter from C to QC. For cvars with complex interactions — sv_aircontrol, sv_airaccel_qw, sv_warsowbunny_* — whether the QC implementation produces identical movement for the same cvar values was not verified from commit history alone. The gravity compensation added in 0.8.6 confirms at least one measurable difference existed post-migration.
? sv_gameplayfix_q2airaccelerate exact semantics. Confirmed present and set to 1 in xonotic-server.cfg. The precise behavioural difference between Q2-style and standard air acceleration was not traced through DarkPlaces C code.
? Pre-2011 history. The full diff history begins at "initial checkin from nexuiz svn r8756" but the Nexuiz-era values and any changes in 2009–2011 were not traced further here.
