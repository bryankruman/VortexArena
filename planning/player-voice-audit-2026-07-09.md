# Player voice / taunt audio audit — 2026-07-09

Audit of every human-recorded player voice line (taunts, team-shoot callouts, comms-wheel
messages, death sounds) shipped in the Xonotic asset set, to identify offensive content for
removal. Four lines containing an ableist slur were removed; a set of strong-profanity taunts
is flagged for a follow-up product decision.

> **Where the assets live.** The player voice `.ogg` files and the `.sounds` manifests that
> index them are **not** in this repo — they live in the Xonotic asset tree
> (`<data>/xonotic-data.pk3dir/sound/player/…` and `…/models/player/*.iqm_0.sounds`), which is
> mounted into the game at runtime via the `assets/data` junction (`XG_DATA_DIR`). This document
> lives in the port repo as the durable record; the actual audio/manifest edits were made in the
> asset tree. See **Changes made** for the exact file list so the same edits can be reproduced
> against any fresh copy of the assets.

---

## How player voice lines work (so future edits don't break)

- Each player model has a `.sounds` **manifest** (`models/player/<model>.iqm_0.sounds`) mapping a
  voice **id** (`taunt`, `teamshoot`, `death`, `pain50`, …) to a sample path and a **variant
  count** `N`:

  ```
  teamshoot sound/player/marine/coms/teamshoot 4
  ```

- The host resolver ([`game/loaders/PlayerSoundResolver.cs`](../game/loaders/PlayerSoundResolver.cs))
  mirrors QuakeC `LoadPlayerSounds`: for count `N > 0` it picks a **random** variant among
  `{path}1 … {path}N`. Count `0` means a single file at the exact path.
- **Consequence for removal:** you cannot simply delete `teamshootK.ogg`. If a file in the middle
  of the `1..N` range goes missing, the resolver still rolls that index ~`1/N` of the time and the
  callout plays **nothing**. A safe removal must (a) delete the offending file, (b) **renumber**
  the surviving higher-indexed files down so `1..N` stays contiguous, and (c) **decrement `N`** in
  **every** manifest that references that sound pack.
- Multiple models share one sound pack. The mapping (model → pack, via `//TAG:` and the path):

  | Sound pack     | Referenced by model manifest(s)                         |
  |----------------|---------------------------------------------------------|
  | `soldier`      | `erebus`, `megaerebus`                                   |
  | `reptilian`    | `gak`, `gakmasked`                                       |
  | `insurrectionist` | `ignis`                                              |
  | `marine`       | `ignismasked`                                           |
  | `specop`       | `nyx`, `seraphinamasked`, `umbra`                       |
  | `pyria-skadi`  | `pyria`, `seraphina`                                     |
  | `carni-lycan`  | *(none — referenced only in commented-out template lines)* |
  | `torus`, `robot`, `fricka` | *(present in asset tree; not wired via the current model manifests)* |

- The default fallback pack is `sound/player/default.sounds`, an **espeak TTS** pack (generated by
  `sound/player/espeak/make.sh`). Its text is known and inoffensive (taunt = "Double facepalm!").

## Methodology

The lines are audio only — no transcripts ship with the assets. All 200 human-recorded
`coms/*.ogg` and `player/death*.ogg` files were transcribed with OpenAI Whisper (`base` model for
the full sweep, `small` model to re-confirm the flagged team-shoot files). Pain/jump/fall grunts
and death screams are non-verbal and were spot-checked only. Whisper softens the slur in the
flagged files to "retired"/"Richard", but the `small`-model pass plus manual listening confirmed
the actual word in all four.

Transcription accuracy caveats: very short or noisy clips occasionally produce hallucinated or
wrong-language output (e.g. `carni-lycan/coms/seenflag` → "I said the fuck!" is a misread of an
accented "I see the flag"). Such cases were manually verified before any action.

## Changes made (removed — ableist slur "retard")

Four team-shoot callouts contained the slur. Removed from the asset tree, survivors renumbered,
manifest counts decremented `4 → 3`:

| Pack | File removed | Transcript | Renumbering | Manifest count |
|------|--------------|-----------|-------------|----------------|
| `carni-lycan` | `coms/teamshoot3.ogg` | "Same team retard!" | none (was last index) | n/a — pack unreferenced |
| `marine` | `coms/teamshoot3.ogg` | "Same team, retard!" | `teamshoot4 → teamshoot3` ("…idiot") | `4 → 3` |
| `pyria-skadi` | `coms/teamshoot2.ogg` | "You retard, we're on the same team!" | `3 → 2`, `4 → 3` | `4 → 3` |
| `specop` | `coms/teamshoot2.ogg` | "You retard, we're on the same team!" | `3 → 2`, `4 → 3` | `4 → 3` |

**Manifests edited** (`teamshoot … 4` → `… 3`): `ignismasked` (marine); `pyria`, `seraphina`
(pyria-skadi); `nyx`, `seraphinamasked`, `umbra` (specop). No port code or player config change
was needed — variant counts are read only from the runtime `.sounds` manifests; nothing in the
port repo hardcodes them.

**Validation:** for every pack referenced by a manifest, files-on-disk == declared count (3); no
orphaned `teamshoot4.ogg` remain; `carni-lycan` (unreferenced, lost only its last index) cannot
mismatch.
