# Parity drift — Waves 16/17 consolidated re-verify (2026-06-27)

**112 units re-audited (12 + 100) · 0 failed · 387 gap-dimensions confirmed closed this pass · 0 regressions.**

This report covers the opus re-audit of every unit touched by the two final porting waves:
- **Wave 16 — state producers** (66 units, 244 edits) — commit `b3a9c06`
- **Wave 17 — presentation consumers** (46 units, 134 edits) — commit `2f0bfdb`

The in-wave verify phase was rate-limited at the tail of both runs, so re-verification was completed afterward
by the chunked `_verify-only.workflow.js` harness (12 units in `44252f3`, the remaining 100 here). Each unit's
`registry/<unit>.yaml` was re-scored against the current port (skeptical rules: coded-but-uncalled ⇒
liveness:partial/dead, not closed; unverified visible fidelity ⇒ `unknown`, not `faithful`).

The "features with gaps" drop (−59) is smaller than the 387 closed *dimensions* because (a) most closures were
*secondary* dimensions of features that still carry a worst-dimension gap, (b) **357 gaps were honestly flagged
`unportable`** (real feature/render builds — see below) and stay open, and (c) several re-scores moved
`missing → unknown/partial` (still a gap, but no longer "absent"). The two numbers that matter most — absent
logic (−24) and absent visuals (−22) — are the real parity wins: **46 features that had no core behaviour or no
visual now have one.**
