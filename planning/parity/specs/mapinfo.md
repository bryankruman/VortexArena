## Overview

MapInfo is Base's per-map metadata subsystem: it parses (or **autogenerates**) `maps/<bsp>.mapinfo`
files to learn each map's title/author/description, which gametypes it supports (plus per-gametype
default limits and per-map cvar overrides), feature requirements (weapons/turrets/vehicles/monsters),
visibility flags (hidden/forbidden/frustrating/donotwant), music (`cdtrack`), fog, and per-map settemps.
Every map-facing surface consumes it: the menu map list + map-info dialog, the server map rotation,
the map/gametype vote availability checks, `lsmaps`, and ÔÇö critically ÔÇö **worldspawn boot**, where
`MapInfo_LoadMapSettings` decides whether the configured gametype can run on this map and falls back
to a supported one when it can't. The known tdm/clanarena/duel "map support" gaps recorded in those
units are symptoms of this subsystem.

## Base algorithm (authoritative)

### .mapinfo parsing (`MapInfo_Get_ByName_NoFallbacks`, mapinfo.qc:1035-1381)
- **Entry:** `MapInfo_Get_ByName(name, allowGenerate, gametypeToSet)`; cache-retrieve shortcut only
  when `gametypeToSet == NULL` (ApplyGametypeEx has cvar side effects).
- Rejects `/` in the name. Opens `maps/<name>.mapinfo`; if absent and `g_mapinfo_q3compat == 1`
  tries `.arena`/`.defi`; else `maps/autogenerated/<name>.mapinfo`; else (if allowed) **generates**
  one (returns 2 in that case, so list builders can yield a frame).
- Line keywords: `title` `description` `author` `has (weapons|turrets|vehicles|monsters|new_toys)`
  `hidden` `forbidden` `frustrating` `donotwant|noautomaplist` `gameversion_min` (Ôćĺ DONOTWANT when
  `gameversion` too old) `type` (legacy positional) `gametype <mode> [key=valueÔÇŽ]` `size x1 y1 z1 x2 y2 z2`
  (mins<maxs guard) `settemp_for_type (all|mode) cvar value` `clientsettemp_for_type ÔÇŽ` `fog ÔÇŽ`
  (cvar_value_issafe-guarded) `cdtrack <t>` (Ôćĺ `MapInfo_Map_clientstuff += cd loop "t"` only when a
  gametype is being set). Comments: `#`, `//`, `_` prefixes + trailing `//`.
- **SVQC fallback** (1347-1358): when parsing the currently loaded map, missing title ÔćÉ `world.message`,
  missing author ÔćÉ worldspawn `author` key.
- Post: `MapInfo_title_sans_author` (separators `" by "`, `" (by "`, `" [by "`, `"Made By "` promote
  the suffix to author), `MapInfo_isRedundant` (normalized bspname==title) picks the titlestring form
  `^2bsp` / `^2title` / `^2bsp ^7// ^2title`; `<AUTHOR>` displays as empty. Cache-store. A map that
  ends with **zero supported gametypes is rejected** (treated as "supports no gametypes, ignored").
- `MapInfo_Get_ByName` then dispatches `m_isForcedSupported` for every gametype (duel unless
  `g_duel_not_dm_maps`; tdm when `g_tdm_on_dm_maps`; mayhem/tmayhem/tka/survival variants) via
  `_MapInfo_Map_ApplyGametypeEx("", ÔÇŽ)` ÔÇö i.e. forced support also applies that mode's defaults.

### Gametype defaults + overrides (`_MapInfo_Map_ApplyGametypeEx`, mapinfo.qc:543-618)
- Runs when the parsed gametype matches the one being set. Resets `timelimit`/`leadlimit`/`fraglimit`
  to cvar defaults, clears every gametype's `m_parse_mapinfo` state, then walks
  `"<gametype defaults (Gametype.model2)> <mapinfo line args>"` as `key=value` tokens:
  `timelimit`, `leadlimit` set directly; `pointlimit|fraglimit|lives|laplimit|caplimit` Ôćĺ fraglimit
  (normal), `teampointlimit|teamlaplimit` Ôćĺ fraglimit when `g_race_teams >= 2`; anything else is
  offered to every gametype's `m_parse_mapinfo(k,v)` (e.g. `teams=` Ôćĺ `g_tdm_teams`). Unknown keys warn.
- The legacy `type` form (`_MapInfo_Map_ApplyGametype`) is positional
  (`fraglimit timelimit [teams] [leadlimit]`, Race special-cases qualifying timelimit + team fraglimits).

### Autogeneration (`_MapInfo_Generate`, mapinfo.qc:281-441)
- Reads `maps/<map>.ent` else the `.bsp` entity lump. Worldspawn: captures `author`, `_description`,
  `music`/`noise`, `message` (title). Entities: bbox of all `origin`s Ôćĺ `diameter = vlen(maxs-mins)`;
  counts spawnpoints (`info_player_team1/team2/start/deathmatch`); `race_place > 0` Ôćĺ spawnplaces;
  classname prefixes set features (`weapon_*` Ôćĺ WEAPONS [nex/railgun excluded], `turret_*`, `vehicle_*`,
  `monster_*`); `target_music|trigger_music` clears worldspawn music; `target_stopTimer` marks a q3df
  map (skip standard-mode heuristics); every other classname is offered to `m_generate_mapinfo`.
- Support heuristics: if a two-base mode (`m_isTwoBaseMode`: as/ctf/nb/rc) is already supported Ôćĺ
  do NOT auto-add base-less modes; else (non-q3df) OR-in every gametype whose
  `m_isAlwaysSupported(spawnpoints, diameter)` passes (typical: `spawnpoints >= 8 && diameter > 3250`
  for dm-family/team modes; duel: `diameter < 3250`). RACE without `race_place` demotes to CTS.
- Writes `maps/autogenerated/<map>.mapinfo`: title/description/author, `cdtrack` (worldspawn music
  minus `.wav/.ogg` suffix, else a **random** track index 1..n from `g_cdtracks_remaplist` excluding
  `g_cdtracks_dontusebydefault`), `has` lines, `frustrating`, one `gametype` line per supported mode
  with its defaults as a comment.
- With `g_mapinfo_q3compat >= 2`, `.arena`/`.defi` files feed the generator first (`== 3` skips the
  entity scan when they succeed).

### .arena/.defi parsing (`_MapInfo_ParseArena`, mapinfo.qc:820-970)
- Multi-map aware ({}-blocks matched on `map <name>`, case-insensitive). Fields: `longname`Ôćĺtitle,
  `author`, `quote`Ôćĺdescription, `type` (Q3 gametype list: `team`Ôćĺ`tdm ft`, `ffa`Ôćĺ`dm lms ka`,
  `tourney`Ôćĺ`duel`; team maps without duel get `+ ca kh`; sets FEATURE_WEAPONS), `.defi` `style` Ôćĺ CTS.
- `_MapInfo_FindArenaFile`: `scripts/<map>.arena` first, else glob `scripts/*.arena` **inside the map's
  own pk3** (search_packfile).

### Filtering + enumeration (`MapInfo_Enumerate` / `_MapInfo_FilterGametype` / `MapInfo_FilterString`, qc:134-246)
- Enumerate: glob `maps/*.bsp`, invalidate cache. FilterGametype: for each map, Get_ByName(allowGenerate)
  ÔÇö if one was generated and pAbortOnGenerate, bail and resume next frame (`MapInfo_progress`);
  keep maps where supportedGametypes Ôłę wanted, features ÔŐç required
  (`MapInfo_CurrentFeatures()`: WEAPONS unless instagib/overkill/nix/weaponarena/!pickup_items/
  melee_only/race/cts/nexball/ca/freezetag/lms), forbidden flags clear, required flags present
  (`MapInfo_ForbiddenFlags()`: FORBIDDEN always + HIDDEN unless `g_maplist_allow_hidden` + FRUSTRATING
  unless `g_maplist_allow_frustrating`; `MapInfo_RequiredFlags()`: FRUSTRATING when that cvar > 1).
  Heapsort by strcasecmp. FilterString: keyword filter on lowercased title (bspname when title unset).

### Map-play checks (`MapInfo_CheckMap` / `MapInfo_LoadMapSettings` / `MapInfo_LoadMap`, qc:1489-1628)
- `_MapInfo_CheckMap(s, gametype_only)`: Get_ByName ok Ôłž supports `MapInfo_CurrentGametype()`
  (Ôłž features ÔŐç CurrentFeatures unless gametype_only). Consumers: rotation `Map_Check`, vote/suggest
  validation, lsmaps, GotoNextMap.
- `MapInfo_CurrentGametype()`: any `g_<mode>` cvar set that differs from `gamecfg`'s mode wins, else
  gamecfg, else DM.
- `MapInfo_LoadMapSettings(s)` (worldspawn): save current gametype (`MapInfo_SwitchGameType` fans out
  all `g_*` cvars + sets `gamecfg`, latches `MapInfo_LoadedGametype`). If the map does NOT support it:
  `g_mapinfo_allow_unsupported_modes_and_let_stuff_break` Ôćĺ keep mode, apply override settings only;
  total mapinfo failure Ôćĺ random **preferred** (m_priority==2) mode; else **RandomSelection over the
  map's supported modes weighted by m_priority** Ôćĺ switch to it. Then Get_ByName(s, 1, chosenType)
  applies that gametype's defaults + map overrides + settemps + clientstuff.
- `MapInfo_LoadMap`: `cvar_settemp_restore()` then `map`/`changelevel`.
- `MapInfo_FindName(prefix)`: binary search, unique-prefix completion (ambiguous Ôćĺ -1 with first match);
  `MapInfo_FixName` returns the completed name.
- `MapReadSizes(map)`: `maps/<map>.sizes` Ôćĺ `map_minplayers` / `map_maxplayers`.

### Settemps + ACL (`_MapInfo_Parse_Settemp`, qc:666-768)
- `settemp_for_type (all|mode)` applies `cvar_settemp` server-side when the mode matches the one being
  set; `clientsettemp_for_type` appends `cl_cmd settemp` lines to `MapInfo_Map_clientstuff` (stuffed to
  clients at connect). `#include <file>` recurses 1 level. Every name/value must pass
  `cvar_value_issafe`, the system ACL `-g_mapinfo_* -rcon_* -_* -g_ban* +*`, AND the user ACL
  `g_mapinfo_settemp_acl` (server default `-*` = deny) ÔÇö pk3 content must not reconfigure the server.

### Gametype registry (`CLASS(Gametype)`, mapinfo.qh:33-149)
- Per-mode metadata: `netname` (g_ cvar), `mdl` (short name), `message` (pretty name), `m_description`,
  flags TEAMPLAY/USEPOINTS/PREFERRED/PRIORITY(Ôćĺ m_priority 2/1/0)/HIDELIMITS/WEAPONARENA/1V1,
  `m_mutators`, `model2` (default limits string), `m_legacydefaults`, and the m_parse_mapinfo /
  m_generate_mapinfo / m_isTwoBaseMode / m_isAlwaysSupported / m_isForcedSupported hooks.
  32-slot registry; `_GametypeFlags_FromGametype` maps id Ôćĺ a 3-vector bitmask (24 bits/component).
  `MapInfo_Type_FromString` remaps deprecated names (nexballÔćĺnb, freezetagÔćĺft, keepawayÔćĺka,
  invasionÔćĺinv, assaultÔćĺas, raceÔćĺrc, ffaÔćĺdm, cctf/oneflagÔćĺctf, tourneyÔćĺduel, arenaÔćĺca [q3compat]).
  `MapInfo_Type_ToText(NULL)` = "@!#%'n Tuba Throwing".

## Parity assessment

- **Liveness headline:** the MENU half of MapInfo is genuinely live (two parsers, the dialog and the
  maplist filter, both invoked every create-game session ÔÇö this supersedes tdm.yaml's stale
  "MapInfoBackend dead" note). The SERVER half is a facade: every authority-path consumer
  (rotation Init/Map_Check, Voting.ValidateMap, suggestmap) calls predicates that default to `true`
  and are never wired, and worldspawn boot never consults map support at all.
- **Logic:** the biggest divergences are (1) no gametype-support check or priority-weighted fallback
  at map load; (2) the per-map `gametype <mode> key=value` defaults machinery missing;
  (3) autogeneration + support heuristics missing (maps without .mapinfo = unknown support treated
  as supports-all in the menu); (4) flags/features never enforced; (5) mapinfo settemps/fog/clientstuff
  never parsed (with the security ACL absent too ÔÇö must ship together).
- **Values:** the constants that exist match (force-cvars polarity, duel 3250 diameter, remap table,
  glob/sort). The missing subsystems carry their constants with them (alias table, ACL strings,
  heuristic thresholds, cdtrack random-pick).
- **Intended divergences:** in-process cdtrack resolution instead of `clientstuff` stuffing;
  per-world cvar store instead of settemp-restore-on-changelevel; C# dictionary caches instead of
  DP db/buf. Each is recorded on its row with rationale.
- **Cross-references:** tdm.yaml `tdm.rules.map_support`, clanarena.yaml `clanarena.mapinfo.always_supported`,
  duel.yaml `duel.mapsupport.gating`, survival.yaml notes (per-mode symptoms of `generate.always_supported` /
  `forced_supported`); sv-mapvoting.yaml `gametype_vote` (maplist-reset + availability-filter residuals
  = `list.allowed_maps` / `check.checkmap` here); sv-intermission.yaml (rotation machinery);
  sv-campaign.yaml (settemp application + SwitchGameType seam); sv-world-rules.yaml (MapInfo_Shutdown,
  worldspawn boot); sv-commands-votes.yaml (`make_mapinfo` verb missing); race.yaml/lms.yaml/keepaway.yaml/
  freezetag.yaml/invasion.yaml/nexball.yaml (the per-mode "-1 = use mapinfo default" value rows).

## Open questions

- Should the server-side catalog reuse `MapInfoBackend` (Godot-free, injectable reader) wired to the
  server VFS? It already parses enough for `MapSupportsGametype`; wiring it to `MapRotation`'s two
  predicates + `GameWorld` boot would close the three worst gaps without new parsing code.
- Does any shipped official pk3 rely on `settemp_for_type` (balance overrides) on stock maps? If yes,
  its absence changes stock-match balance on those maps and should be prioritized with the ACL.
- The menu treats no-.mapinfo maps as supports-everything; Base autogenerates. Decide whether the port
  should conservatively hide such maps per-gametype or implement the entity-scan heuristics.
