# Upstream Watch — Ledger

The running index of every triaged upstream contribution. One row per item. See
[`README.md`](README.md) for the process, the rubric, the ID scheme, and the vocabularies.

Columns: **UW** id · **Contribution** · **Relevance** · **Decision**. The Contribution cell stacks a
**bold one-sentence headline**, then a ≤6-sentence summary of the change, then a metadata line: the
`source` dedup key (linked to the original **GitLab commit/MR**), the kind, effort, and a deep-dive
link where one exists.

- **UW** — permanent id, never reused; deep dives live in [`items/`](items/). Ids are non-contiguous:
  Transifex/translation autosync commits are filtered out and never get a row.
- **Contribution** — source key = `data@<sha>` / `dp@<sha>` (master commit) or `repo:branch@<tip>`
  (branch/MR). Kind ∈ `qc-gameplay` · `data-cfg` · `asset` · `dp-engine` · `build/i18n/ci`.
- **Relevance** (auditor's port-worthiness rollup) — 🟢 High · 🟡 Medium · 🟠 Low · 🔴 None.
- **Decision** — ⏳ Pending · ✅ Port · 🔧 Adapt · 📦 Ported · ⏸️ Defer · ❌ Reject · ➖ N/A.
